1. **Начало процесса**:
   - Событие: Аналитик инициирует процесс голосом (например, с помощью кнопки "Начать запись").

2. **Запись голоса**:
   - Действие: Система начинает запись голоса аналитика.
   - Условие: Запись продолжается до тех пор, пока аналитик не завершит ее (например, произнесет "Стоп").

3. **Обработка речи**:
   - Действие: Система обрабатывает записанный голосовой ввод и преобразует его в текст.
   - Условие: Проверка на наличие ошибок в распознавании речи.

4. **Генерация диаграммы**:
   - Действие: На основе текстового описания процесса система автоматически генерирует диаграмму процесса.
   - Условие: Диаграмма успешно создана.

5. **Отображение диаграммы**:
   - Действие: Система отображает сгенерированную диаграмму на экране аналитика.

6. **Взаимодействие с аналитиком**:
   - Действие: Система переходит в режим чата, позволяя аналитику вносить правки в диаграмму.
   - Условие: Аналитик может вносить изменения, задавать вопросы или запрашивать уточнения.

7. **Внесение правок**:
   - Действие: Система принимает изменения от аналитика и обновляет диаграмму в реальном времени.
   - Условие: Проверка на корректность внесенных правок.

8. **Завершение процесса**:
   - Событие: Аналитик завершает работу с диаграммой (например, произнесет "Завершить" или нажмет кнопку "Сохранить").
   - Действие: Система сохраняет финальную версию диаграммы и завершает процесс.

9. **Конец процесса**:
   - Событие: Процесс завершен.

Эта последовательность шагов может быть использована для создания BPMN-диаграммы, которая визуализирует процесс разработки сервиса для формирования диаграмм процессов на основе голосового ввода. Каждый шаг может быть представлен в виде элементов BPMN, таких как события, действия и условия.
